# Selected Direction · AI 职业赚钱路径诊断器

**Gate**：G2
**日期**：2026-04-24
**决定**：**O1 单页诊断网页**作为 v0.1 唯一方向。

---

## 1. 一句话定位

> 为"想用 AI + 内容 + 网页产品赚钱"的个人，提供一个 5 分钟填完就能拿到"7/30/90 天可执行行动清单"的网页诊断工具。

---

## 2. 为什么是这个方向

三个关键理由：

1. **跑通系统本身比赚钱重要**。O1 是唯一能同时验证"调研 + 战略 + 产品 + 工程 + 测试 + 增长 + 沉淀"七个环节的项目形态。它是本 AI Venture OS 的第一个"完整样板"。

2. **个人资源最匹配**。纯前端、无数据库、无登录 → Claude Code + GitHub Pages 就能跑通 → 无成本实验。

3. **天然有漏斗顶端价值**。哪怕商业上只服务 1000 人，只要这 1000 人中有 30 人想付费深化，就能直接接 O2（模板包）和 O3（定制报告）。

---

## 3. 关键约束（v0.1 不可动摇）

- 单页（没有路由、没有多页）
- 纯前端（没有服务端、没有 API 调用、没有数据库）
- 规则化输出（不是实时 LLM 生成）
- ≤10 个输入字段
- 移动端优先
- 一键复制 Markdown 必须有
- 代码全部保持可读，不引框架（HTML + 原生 JS + 少量 CSS）

---

## 4. 明确不做清单（锁死边界）

| 诱惑 | 为什么不做 |
|---|---|
| 加登录系统 | 验证阶段登录会损失 50%+ 首次使用转化 |
| 加数据库存历史 | localStorage 足够，库是过度工程 |
| 接 OpenAI / Claude API | 成本 + 不必要，规则化足以验证概念 |
| 做多语言 | 先验证中文主画像，英文是 v0.3 的事 |
| 做"AI 工具商城" | 不是本项目，是另一个项目 |
| 做"用户社群" | 社群是 O7，已否决 |
| 加付费墙 | v0.1 不做商业化，不让钱扭曲数据 |

---

## 5. 后续漏斗（v0.2+）

```
v0.1 免费诊断网页  ← 现在在这里
    │
    ├──→ v0.2  某类职业（比如"独立开发者"）完整 Prompt + SOP 包  ¥29–¥99
    │
    └──→ v0.3  半自动定制深度报告（用 Claude API，人工审阅）     ¥999+
```

但 **v0.1 不跑通前，v0.2/v0.3 都不动工**。

---

## 6. 唯一成功标准（简化）

用户测试中，**≥3 人**看到报告后说："这个我想发给我朋友 / 我想明天就照这个做"。

达不到 → 整个产品回到 Gate 1 重审。
达到了 → 进 v0.2。
