# User Flow · AI 职业赚钱路径诊断器 v0.1

**Gate**：G3

---

## 主流程（Happy Path）

```
[访问页面]
    │
    ▼
[看到 Hero + 1 句说明]  ←── 10 秒内必须看懂"输入信息，给你 AI 赚钱路径"
    │
    ▼
[向下滚动 / 点击"开始诊断" CTA]
    │
    ▼
[9 字段表单]
    │  ├ 职业 (单选 + 其他)
    │  ├ 技能 (多选 tag，≤3)
    │  ├ 时间 (单选)
    │  ├ 目标收入 (单选)
    │  ├ 风险偏好 (单选)
    │  ├ 愿意做内容？
    │  ├ 会写代码？
    │  ├ 愿意做咨询？
    │  └ 愿意做产品？
    ▼
[点击"生成我的路径" 按钮]
    │
    ▼
[页面无刷新，报告区域展开 + 滚动到报告顶部]
    │
    ▼
[阅读 8 模块报告]
    │
    ├──→ [点击"复制 Markdown"] ──→ [按钮变"已复制✓"] ──→ 粘贴到 Obsidian/笔记
    │
    ├──→ [向下滚到底部 CTA] ──→ 下一步建议（未来 v0.2 付费/订阅/社群入口，v0.1 只放占位）
    │
    └──→ [返回表单修改再生成]
```

---

## 首次 vs 再次访问

| 场景 | 行为 |
|---|---|
| 首次访问 | 表单为空，所有字段待填 |
| 再次访问 | 表单回填上次输入（localStorage），CTA 改为"重新生成"或"修改再试" |

---

## 异常分支

| 场景 | 处理 |
|---|---|
| 必填字段未填 | 按"生成"时高亮未填字段，滚到第一个未填位置 |
| 职业选"其他" | 文本输入框出现，生成时走 `general` 兜底规则 |
| 技能选 0 项 | 阻塞，提示"至少选 1 项" |
| 网络断开 | **不受影响**（纯前端），正常生成 |
| 剪贴板写入失败 | 降级：弹出带 `<textarea>` 的模态，用户手动选中复制 |

---

## 用户视角成功路径

1. 被朋友/自己搜索转发进来 → 3 秒内理解产品
2. 花 3–5 分钟填表 → 有些字段甚至想笑（因为问得具体）
3. 生成报告 → 看到引用了自己输入字段的具体建议，感觉"这不是套话"
4. 复制 Markdown → 粘贴到自己的笔记工具里
5. 对一条 7 天行动计划说"行，我明天就试"
6. 未来转发给一个朋友

这 6 步中任一断裂，v0.1 就失败。
