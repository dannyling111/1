## 1. 一句话

输入你的职业 + 技能 + 可投入时间 + 目标收入 + 风险偏好，页面立即生成"AI 时代你的赚钱路径"结构化报告，支持一键复制 Markdown。

---

## 2. 技术栈（锁死，不允许扩展）

- **前端**：原生 HTML + 原生 JS + 原生 CSS。**不引入任何框架**。
- **样式**：单一 `<style>` 块，移动端优先，不用 Tailwind/Bootstrap。
- **存储**：仅 `localStorage` 保存用户最近一次输入（方便回来再填）。
- **外部依赖**：**零**。不用 CDN，不加字体文件，不加埋点第三方。
- **部署目标**：GitHub Pages（默认）；代码结构要兼容 Vercel / Cloudflare Pages 直接上。
- **文件结构**：
  ```
  app/
    index.html        # 唯一页面
    style.css         # 可选，也可嵌进 index.html
    app.js            # 规则引擎 + 报告生成
    rules/
      careers.json    # 职业库 + 每个职业的 AI 影响规则
      pathways.json   # 3 条赚钱路径模板
      actions.json    # 7/30/90 天行动模板
  ```
  如果 JSON 管理起来麻烦，也可以直接放进 `app.js` 的常量里（v0.1 允许）。

## 3. 用户输入字段（共 9 个，≤10 约束）

| # | 字段 | 类型 | 必填 | 选项/说明 |
|---|---|---|---|---|
| 1 | 当前职业 | 单选 + "其他…" 输入框 | 是 | 预置 15 类（见下文§5.1）；"其他"时用 `general` 兜底规则 |
| 2 | 核心技能（最多 3 项） | 多选 tag | 是 | 写作、视觉、代码、数据、营销、咨询、运营、设计、产品、研究、销售、工程 |
| 3 | 每周可投入时间 | 单选 | 是 | <5h / 5–15h / 15–30h / 30h+ |
| 4 | 目标月收入（当前币种） | 单选 | 是 | <¥5k / ¥5–20k / ¥20–50k / ¥50k+ |
| 5 | 风险偏好 | 单选 | 是 | 保守（稳定副业） / 平衡（边做边试） / 激进（全职创业） |
| 6 | 是否愿意做内容 | 是/否 | 是 | — |
| 7 | 是否会写代码 / 愿意用 Claude Code | 是/否 | 是 | — |
| 8 | 是否愿意做 1v1 咨询 | 是/否 | 是 | — |
| 9 | 是否愿意做产品 / 网站 | 是/否 | 是 | — |

**摩擦减少**：
- 默认全部用单选或标签按钮，不是文本框。
- 字段顺序从具体到抽象（先问职业，最后问偏好）。
- 没有 "Next"，单页分段滚动显示。
- 表单可以分成 3 段视觉分区，但不分步，减少跳转。

## 4. 报告输出（8 模块，顺序固定）

生成规则严格结构化。每个模块必须产出，**不允许留空**。

1. **你的 AI 替代风险**（1 段 + 0–100 评分）
   - 基于"当前职业 × 核心技能"的规则匹配
   - 同职业不同技能组合结果不同（保证同职业用户之间不一样）

2. **你的 AI 增强机会**（3 条）
   - 写清"加上 AI 后，你能做什么原来做不了的事"

3. **最适合的 3 个赚钱方向**（排序）
   - 每条包含：方向名 + 一句话解释 + 匹配理由（结合输入字段）
   - **不允许**出现"学 Prompt 工程"这种没卡点的建议
   - 建议按输入第 5/7/8/9 项切分逻辑（见 §5.2）

4. **7 天行动计划**（3 条，每条带"今天就能做"的具体动作）

5. **30 天行动计划**（3 条）

6. **90 天行动计划**（3 条）

7. **推荐工具栈**（≤7 个工具，每个工具配一句"你用来做什么"）

8. **下一步：3 件明天就能做的事**（3 条短小可执行）

## 5. 规则引擎设计

### 5.1 职业库（15 类初始）

程序员、产品经理、设计师、运营、市场营销、销售、内容创作者、视频博主、自媒体作者、咨询顾问、财务/会计、律师、教师/培训师、研究/学术、其他。

每个职业预置：
- `ai_risk_base`：0–100 基础替代分
- `ai_augment_examples`：3 条增强场景模板
- `typical_pathways`：可适用的赚钱路径 ID 列表

### 5.2 赚钱路径库（初始 9 条）

| ID | 路径 | 触发条件 |
|---|---|---|
| P1 | 做内容（博客/视频/小红书） | §3.6 = 是 |
| P2 | 做 AI 增强的自由职业 | 技能含 代码/设计/写作/营销 |
| P3 | 做 1v1 咨询 / 定制服务 | §3.8 = 是 |
| P4 | 做 Prompt/SOP 包低价产品 | §3.7 = 是 且 §3.9 = 是 |
| P5 | 做单页 Web 工具（免费引流） | §3.7 = 是 且 §3.9 = 是 |
| P6 | 做垂直信息聚合 Newsletter | §3.6 = 是 且 §3.5 = 平衡/激进 |
| P7 | 做 AI 增强的 B 端服务 | 技能含 咨询/营销/销售 |
| P8 | 做付费社群 / 小圈子 | §3.5 = 激进 且 §3.6 = 是 |
| P9 | 做在职 AI 提效副业 | §3.3 < 15h/周 且 §3.5 = 保守 |

### 5.3 匹配算法（可以极简）

```
1. 从 careers.json 查到用户职业的 ai_risk_base 和 typical_pathways。
2. 技能、时间、偏好等字段做加/减分修正（+10~-20）。
3. 从 typical_pathways 里筛出满足"触发条件"的路径，按匹配分排序。
4. 取 Top 3 作为 §4.3 输出。
5. 7/30/90 天行动计划按 Top 1 路径展开，用 actions.json 模板填。
```

**关键**：相同职业的不同用户，因为技能/时间/偏好不同，报告必须明显不同。**不能出现两个不同用户看到同一份报告**。这是规则引擎的正确性测试点。

### 5.4 "输出不像套话"的设计

每个输出字段里必须引用用户输入里的至少 2 个变量。例子：

❌ 坏：「你可以做自由职业，用 AI 提高效率」
✅ 好：「作为【设计师】+【每周 15–30h】，先把你的 Figma 工作流用 Claude Code 接一个自动排版脚本；7 天目标：拆出 3 个重复动作，做成可复用 Prompt」

PRD 强制：**每段建议必须引用至少 2 个用户输入字段的值**。

## 6. 一键复制 Markdown

复制按钮点击后：
1. 调用一个 `generateMarkdown(reportData)` 函数
2. 生成固定结构的 Markdown（见下）
3. 用 `navigator.clipboard.writeText(md)` 写入剪贴板
4. 按钮文案变 "已复制 ✓" 2 秒

Markdown 格式：
```markdown
# AI 职业赚钱路径诊断报告

> 职业：XXX｜技能：A, B, C｜时间：15–30h/周｜目标：¥20–50k
> 生成时间：2026-xx-xx

## 你的 AI 替代风险
分数：62/100
<说明文>

## AI 增强机会
- ...

## 最适合的 3 条赚钱路径
### 1. ...

...（其余 8 个模块）
```

Obsidian 兼容：用 `##` 不用 `#`（除首行外），每段之间留空行，不带 HTML。

## 7. UI / UX 要求

- 打开页面 10 秒内用户必须知道这是干嘛的（Hero 标题 + 1 句副标题）
- 输入表单总高度控制在 3 屏以内
- 生成后报告不跳页，直接滚到报告区
- 报告区右上角固定一个"复制 Markdown"按钮
- 手机端字号最小 16px，按钮 44px 高
- 无 loading 动画（规则化瞬间生成），避免假装"AI 在思考"的表演
