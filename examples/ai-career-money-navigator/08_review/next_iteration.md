# Next Iteration · AI 职业赚钱路径诊断器 v0.2 规划

**Gate**：G7
**日期**：2026-04-24
**前提**：v0.1 上线 ≥4 周 + 有 ≥100 次真实使用后才启动 v0.2。

---

## 1. v0.2 要解决的 3 个核心问题

### 问题 1：输出深度不够
**症状**（预期）：部分用户反馈"8 段结构好，但行动太泛"。
**应对**：
- 把 Top 1 路径的 7/30/90 天计划从 3 条/档扩到 5 条/档
- 每条行动后附"如果卡在这一步怎么办"兜底建议
- 引入"可选深化"：用户勾选 1 条行动项，页面展开相关 Prompt 模板

### 问题 2：15 职业覆盖不足
**症状**（预期）：冷门职业用户进来后发现只能选"其他"。
**应对**：
- 扩到 30 职业
- "其他"允许自由文本输入职业名，用技能组合 + 关键词匹配合成 profile（仍是规则化，不调 LLM）

### 问题 3：一次性工具留存低
**症状**（预期）：用户填完就走，不会回来。
**应对**：
- 加入"30 天打卡" localStorage 模式：用户完成一条行动可以勾选，页面显示进度
- 加一个可选邮件提醒（用户主动留邮箱，不是默认收集）
- 把"报告"从单次输出变成"持续的行动板"

## 4. 需要持续追踪的"信号"

| 信号 | 触发动作 |
|---|---|
| 单条行动计划被复制的频次 | 频次高 → 扩展该路径的模板 |
| "其他"职业命中 generalist 兜底的比例 | 如 >40% → 说明职业库还窄，优先扩 |
| 从页面离开前后停留时间 | 很短 → 可能是"生成后就走"，需要留存机制 |
| 用户主动找我问的问题类型 | 数据最稠密的宝藏，每周整理进 `02_research/` |

---

## 5. 从 v0.1 → v0.2 不能再犯的错

1. **不要在 v0.2 又加 N 个"听起来很酷"的功能**。上面 7 个功能已经是上限。
2. **不要为了"AI 味"而接 LLM**。规则引擎稳定是杠杆，不是局限。
3. **不要省 Gate 5 测试**。任何改动都跑一次 `test_engine.js`。
4. **不要跳 Obsidian 沉淀**。每一轮迭代都必须更新 `09_obsidian_export/`。

---

## 6. 触发 "停掉这个项目" 的条件

| 条件 | 应对 |
|---|---|
| 4 周内 <30 次真实使用 | 回 Gate 1 重审用户画像（可能主画像错了） |
| ≥50% 反馈"像套话" | 回 Gate 3 重做输出结构；不改就停 |
| 100 次使用后 0 人询问付费 | 去掉 v0.2 的付费功能，保留免费工具；或整体停 |
| 我自己不再觉得有趣 | 停，转下一个项目；**这是合法的停止理由** |
