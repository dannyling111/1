## 2. 部署方案（推荐顺序）

| 选项 | 优点 | 缺点 | 本项目推荐 |
|---|---|---|---|
| **GitHub Pages** | 免费、仓库内零配置 | 国内访问慢、自定义域名要 DNS 配置 | 🥇 第一选择（社群种子用户多数有梯子） |
| Vercel | 速度快、支持自定义域名、免费额度充足 | 需要登录 Vercel 账号 | 🥈 备选（如果国内测试卡就切这个） |
| Cloudflare Pages | 国内访问也比较稳、免费 | 账号注册流程略长 | 🥉 长期备用 |

**行动**：今天把 `examples/ai-career-money-navigator/app/` 目录以 GitHub Pages 发出去。

```
Settings → Pages → Build and deployment
  Source: Deploy from a branch
  Branch: main (or the develop branch)
  Folder: /examples/ai-career-money-navigator/app
```

或者：把 `app/` 的内容复制到 `docs/` 或根目录，选 `/docs` 或 `/` 作为 Pages 源。

## 4. 冷启动路径

**原则**：不投流、不买量。**只靠内容触达主画像**。

### 第 1 周（种子期）

| 日 | 动作 | 渠道 |
|---|---|---|
| D1 | 发布到 GitHub，产品上线 | — |
| D2 | 写 1 篇"我为什么做了这个"的笔记（500 字），自我介绍产品 | 即刻 + 小红书 + X（Twitter） |
| D3 | 给 10 位熟人（独立开发者 / 创作者）发链接，请试用，收集首 30 秒反应 | 微信 1v1 |
| D4 | 整理 D3 反馈，修必须修的 Bug | GitHub PR |
| D5 | 发第 2 条内容："5 个人试用后，他们问的同一个问题是……" | 即刻 + 小红书 |
| D6 | 在 1 个垂直社群（知识星球 / Discord）发链接 | 社群 |
| D7 | 周复盘，看埋点数据，写下一周计划 | GitHub |

**目标**：D7 结束时有 ≥20 次真实使用。

### 第 2–4 周（扩散期）

- 每周 2 条内容，主题为"某个职业 × AI 赚钱路径"（用工具的实际输出做 case）
- 找 3 个合作伙伴（独立开发者 Newsletter / 博客）换一次 cross-promotion
- 对 v0.1 出现的最大问题做 v0.2 迭代（基于真实反馈）

**目标**：4 周结束时有 ≥100 次使用 + ≥3 人主动转发。

## 5. 不做的事（避免资源浪费）

- ❌ 不投广告（没有变现不该烧钱）
- ❌ 不做 Product Hunt 发布（上来就被淹没在 AI 工具潮里）
- ❌ 不铺满所有社交平台（同一时间只维护 ≤2 个渠道）
- ❌ 不主动找大 V 推荐（第 1 周没数据请不动人）
- ❌ 不接 LLM API（成本焦虑会扭曲验证）
- ❌ 不加邮件订阅 / 付费墙（v0.1 验证产品形态，不验证收入）

## 6. 验证指标（埋点 + 访谈）

埋点用 `localStorage` 做轻量记录（不联网，不上 GA）。记录：

- `first_visit_ts` — 首次访问时间
- `form_submit_ts` — 点"生成"的时间（如有）
- `copy_md_ts` — 点复制的时间（如有）

每周手工从用户问一次："哪个环节最像套话？"
这个问题的答案直接喂给下一轮模板迭代。

---

## 7. 风险与应对

| 风险 | 应对 |
|---|---|
| 前 10 次使用反馈"像套话" | 立刻回 Gate 3，重看非套话规则是不是形式大于实质 |
| 全部流量来自 1 人转发 | 这是好事，说明该人群精准；继续扩散，别慌 |
| 规则化输出被同行模仿 | 模板不是壁垒，**持续迭代的数据反馈循环**才是 |
| 用户问"能不能接 GPT 做更个性化" | 收集需求 → v0.3 接 LLM API；但 v0.1 不做 |
